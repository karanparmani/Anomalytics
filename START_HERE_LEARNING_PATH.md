## 2) How to study each module

For each module folder, use this mini-workflow:

- Start with the `notebook/` (or `notebooks/`) content.
- Review `images/` to understand concepts and intuition.
- Use `data/` for hands-on practice.
- If module-specific coding packages exist (for example `*_ca`, `decision_trees`, `gbm`, etc.), run and modify them.
- Write your own summary in a notes file after each module.

A practical cadence:

- **Day 1–2:** Concepts + notebook walk-through
- **Day 3:** Re-implement examples from memory
- **Day 4:** Mini exercise on the module dataset
- **Day 5:** Revision + 10 quiz questions self-test

## 5) Milestones (to know you're on track)

By the end of:

- **Week 4:** You can code confidently in Python + NumPy + pandas and create visualizations.
- **Week 7:** You can explain descriptive stats, probability, and hypothesis testing.
- **Week 12:** You can build, evaluate, and improve ML models end-to-end.
- **Week 16:** You can deliver an NLP mini-project and explain model trade-offs.

## 7) Daily study template (90 minutes)

- 20 min: theory review
- 40 min: notebook coding
- 20 min: one variation experiment
- 10 min: write takeaways + open questions

Consistency beats intensity—do this daily and track progress weekly.
